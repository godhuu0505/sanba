## Review guidelines

Codex GitHub 連携の自動レビューはこの節を指針として参照する（既定で P0/P1 を重点指摘）。

- **言語**: レビューコメント・要約・指摘の説明はすべて**日本語**で書く（Always write all review comments in Japanese）。
- **シークレットを混入させない**: APIキー・トークン・`.env` の値・PII をコード/ログに残さない。
- **観測性**: 新しい処理にはトレース/ログ/メトリクスを通す。観測できない追加処理は指摘する。
- **本番志向**: ローカル専用の実装やハードコードされたエンドポイント/認証情報を避ける。
- **テスト/型/lint**: 振る舞い変更にテストが伴うか、`ruff`/`mypy`/`tsc` を通せるかを確認する。
- **誤りの優先指摘**: ロジックの誤り・境界条件・並行性・リソースリークを最優先で指摘する。
- 指摘は簡潔に、重大度（P0/P1/P2）を添えて日本語で書く。指摘が無ければ「指摘なし（LGTM）」。
