# 外部の状況・経歴・システム用外部要件（サンプル）

これは Elastic Agent Builder の外部コンテキスト・エージェントが読み込む知識ファイルの例。
**アプリのソースコードだけでは知り得ない外部前提**を記述する。実運用では組織ごとの本物の
外部要件ファイルを `sanba-external-context` インデックスへ取り込む（Phase 1）。

## 経緯 / 背景
- 対象プロダクトは前身システムからの移行案件で、旧システムの締めは会計年度末に固定されている。
- 過去に一度、個人情報の域外移転で監査指摘を受けており、データ所在地の制約が厳しい。

## システム用の外部要件（コードに現れない前提）
- 保持期間: 監査ログは 7 年、会話データは 30 日で失効（法務方針）。
- データ所在地: 本番データは国内リージョンに限定。越境転送は不可。
- 連携先: 基幹の請求システムは夜間バッチのみ受付（オンライン更新不可）。
- 稼働時間: 月末 2 営業日は凍結期間でリリース禁止。

## 運用・組織の事情
- 承認は二段階（現場 → 情報システム部）。緊急時のみ事後承認を許容。
- 問い合わせ一次受けは外部委託で、SLA は営業時間内 4 時間。
