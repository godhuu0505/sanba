問いは、答えを産む。

SANBA は、音声で対話しながら要件を解像度高く引き出すマルチエージェントです。会議の前の五分、 一問ずつ問いかけ、食い違いや抜けをその場で拾い上げ、確定した要件を GitHub Issue まで届けます。

名前の由来は産婆術（Socratic maieutics）。相手の中にすでにある答えを、問いによって 引き出す技法です。このマニュアルは、実際の画面キャプチャ（PC / スマホ）で各操作を追えます。

1SANBA とは

ログインすると「ホーム」が開きます。ここから会話（問答）を始めます。左側のメニューで ホーム・過去の要件一覧・アプリ管理を行き来します。スマホでは同じメニューが 右上の ☰ に畳まれます。

2ログインする

SANBA は Google アカウントでログインします。ログイン画面の インタビューを始める 相当のボタンから Google 認証に進みます。参加リンク経由の場合はログイン不要で入れることも あります（招待・参加するを参照）。

3会話を準備する

ホームで会話を始める前に、対象アプリと話したいテーマを整えます。

  1. 対象のアプリを選ぶ（任意）。登録済みのアプリを選ぶと、そのアプリの語彙や確認観点を 前提に問答が進みます。未選択でも会話は始められます。
  2. テーマ（ゴール）を入力する。「何を決めたいか」を書きます。もやっとした状態でも、 問答の中で少しずつ言葉になります。
  3. 参考資料を添える（任意）。画面キャプチャ・図・ドキュメントを添付すると、 サンバが解析して会話の材料にします。
💡
対象アプリは後からでも変えられます。まずは思いつくテーマで会話を始めてかまいません。

4音声で対話する（問答）

SANBA の中心が、この問答画面です。サンバ（産婆さん）が一問ずつ問いかけ、あなたが声で 答えると、その場で要件が立ち上がり、食い違いや抜けが検知されていきます。

画面の見方

上部の帯に、いまの要件数・ 未解消数・ 参考資料数が並びます。右上の ● REC は録音中の目印と経過時間です。中央のタブで 会話履歴 / 参考資料 / 要件一覧を切り替えます。

サンバの状態

左のサンバのイラストと状態表示で、いまサンバが何をしているかがわかります。

  • 聞いています — あなたの発話を聞き取っている状態。
  • 考え中… — 受け取った内容を複数のエージェントが検討している状態。
  • 発話中 — サンバが問いかけ・応答を話している状態。

話す・入力する

下部のバーで、発話とテキスト入力を操作します。

  • 押して話す — ボタンを押している間だけマイクが入ります。PC では Space でも操作できます。
  • スピーカー消音 — サンバの音声出力を一時的に止めます。
  • テキストで入力 — 声が使えない場面では、テキストでも答えられます。

食い違いと確認事項

会話の中で食い違い（矛盾）や抜け（ギャップ）を見つけると、サンバがその場で拾い上げ、 確認事項ツリーに整理します。要件一覧タブで、確定してきた要件と、未解消／解消済みの 確認事項をまとめて見られます。

参考資料

添付した資料は参考資料タブに並びます。解析が終わると「解析済み・抽出 N 件」のように表示され、 会話の中で「画面に検索バーが無いのに検索したいと言った」といった食い違いの検知にも使われます。

5会話を終える

右上の■（会話を終了）を押すと、終了確認が出ます。未解消の確認事項が 残っていると、その分は確定されない旨が案内されます。会話を続けるで戻るか、 終了するで成果画面へ進みます。

⚠️
未解消を残したまま終えると、その要件は「未確定」として扱われます。後から続きの会話もできます。

6成果を確認する

会話を終えると、その場で成果がまとまります。ぜひ必要／あると助かるで優先度別に要件が 並び、全文を確認するで詳細へ、Issueで GitHub への書き出しへ進めます。

7要件の詳細と Issue 起票

成果の全文（要件一覧）画面では、ゴール・参考資料・優先度・確信度・発言者つきで要件を確認できます。 読み手に合わせて利用者向け / 企画者向け / 開発者向けのドキュメントを出力でき、 紐づく GitHub リポジトリへIssue を作成できます。

🔗
Issue 起票にはアプリと GitHub リポジトリの紐づけが必要です。アプリを管理するで設定します。

8過去の要件を見る

左メニューの過去の要件一覧から、これまでの会話の成果を一覧できます。日付・ラベル・ 起票済みかどうかが並び、選ぶと要件結果の詳細（7 章）を開けます。

9アプリを管理する

左メニューのアプリ管理で、対象プロダクトを登録・設定します。会話の前提となる情報を ここで整えるほど、問答の解像度が上がります。

  • 基本情報 — アプリ名・URL キーワード・説明。
  • 利用者向け語彙 — 画面名や機能の呼び名。利用者との会話ではこの言葉で問いかけます。
  • 要件結果の出力フォーマット — 利用者／企画者／開発者ごとの文書テンプレート。
  • 会話中の確認項目 — 毎回必ず確認したい観点（最大 20 個）。
  • 前提リポジトリ — 紐づけた GitHub リポジトリを索引し、会話の前提に使います。
  • メンバー — メールアドレスで招待し、会話できる人を管理します。
  • 会話リンク — URL を開くだけで会話を始められるリンクを発行します。

10招待・参加する

他のメンバーは、招待を受けて参加します。メール招待のリンクを開くと招待の受諾画面が、 会話リンクを開くと会話への参加（同意）画面が表示されます。参加リンクは、ログイン不要で 利用者が入れる設定にもできます。

11アカウント設定

右下（PC は左下）のアカウントからアカウント設定を開きます。プロフィールの確認、 GitHub 連携（Issue 起票やリポジトリ索引に必要）、ログアウトを行えます。

12デザインについて

SANBA の画面は「白い紙 × 原色 × ステッカー × 棒人間」のデザインで統一されています。 再利用部品のショーケース（UI Kit）は、開発時の /design 画面で確認できます。
