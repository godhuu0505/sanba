# 画面文言一覧（UI コピー洗い出し） — apps/web

> **一時作業文書（非 SoT）** — ADR-0050 原則4。文言整理（コピーの見直し）の**入力**として、
> `apps/web` の実装から表示文言を棚卸ししたスナップショットです。恒久リファレンスではありません。
> コードが正で、本票はコードから離れると陳腐化します。整理が着地したら削除/アーカイブします。
>
> - スナップショット日: 2026-07-07
> - 対象コミット時点: ブランチ `claude/app-copy-inventory-9plpuv`
> - 抽出元: `apps/web/app/**`・`apps/web/components/**`・`apps/web/lib/**`（テスト・コメントを除外）

## この文書について / スコープ

**目的**: 画面に表示される固定文言を「画面 × カテゴリ」で網羅し、文言整理の土台にする。

**含むもの**
- JSX のテキストノード、見出し、ボタン、ラベル、プレースホルダ、`aria-label`、空状態、
  トースト/エラー/確認ダイアログ/バリデーションのメッセージ、ツールチップ
- 定数・配列で定義された表示用ラベル（状態→日本語ラベルのマッピング等）
- テンプレート文字列（動的差し込みは `{変数}` で表記）

**含まないもの（別管理・動的・範囲外）**
- 産婆（LLM）の発話・要件文・生成ドキュメント本文などの**動的生成物**
- サーバ返却の `detail` 文字列そのもの（多くは画面側で日本語に再マップ済み。再マップ後の文言は収録）
- Google 純正サインインボタン内の文言（Google 側が描画）／Google Picker の UI
- `apps/api`・`apps/worker` が返す文言、純粋な内部エラーコード（`drive_not_configured` 等）
- `className`・URL・型名などの非表示文字列

**凡例**
- `` `"文言"` `` = 表示文言（原文ママ。前後の空白・句読点も原文）
- `{変数}` = 動的差し込み箇所
- （…）= 表示条件・補足
- aria = スクリーンリーダー向け（`aria-label` 等。視覚には出ないが読み上げられる）
- **通常 / end_user** = インタビュー・モードによる出し分け（[ADR-0032](../adr/0032-guest-join-and-enduser-mode.md)）。
  通常＝企画者/開発者向け（産婆術の古風な語り）、end_user＝利用者向け（やわらかい語り）

**関連ドキュメント**: 画面ごとの要件票は [`../reference/screens/`](../reference/screens/)、
会話体験は [`../reference/conversation-experience.md`](../reference/conversation-experience.md)、
利用者モードは [`../reference/product-enduser-requirements.md`](../reference/product-enduser-requirements.md)。

## V. 付録 — トーン/語彙のゆれ（整理候補メモ）

> 以下は洗い出しから見えた**整理の観点**（提案であり、抽出結果とは別）。要判断。

1. **プロダクト行為の呼称ゆれ**: `壁打ち` / `要件サンバ` / `深掘り` / `問答` / `インタビュー` が混在。
   - 例: ホーム `"＋ 壁打ちを始める"`、準備 `"要件サンバを始める"`、参加 `"深掘りを開始する"`、会話ヘッダ `"問答"`。
   - 読み手（利用者/企画者/開発者）別に意図的に使い分けているのか、ゆれなのか要確認。
2. **助詞ゆれ**: 会話シェルの表示 `"結果へ戻る"` と aria `"結果に戻る"`。
3. **マイク/音声まわりの語彙**: `消音` / `ミュート` / `集音` / `聞き取り中` / `認識中` が近接概念で混在（BottomBar・VoiceStatusIndicator・VoiceInputBar・Waveform）。
4. **省略記号のゆれ**: `…`（三点リーダ）と `・・・`（中黒3つ、準備画面 placeholder）が混在。
5. **句点の有無**: ボタン/ラベルは無句点、説明文は句点あり、で概ね統一されているが一部要確認。
6. **英語混じり**: 技術例外（S-2）や `"Markdown をコピー"` `"PDF/Drive/Issue"` など。利用者モードでの露出可否を要確認。
7. **絵文字/記号**: `"📦"` `"🔗"` `"✦"` `"⎋"` `"⎘"` `"✕"` `"✓"` `＋` など。トーンとの整合を要確認。
8. **端末/環境依存の既定値**: `StatusBar` の `"9:41"` `"5G  100%"` はモック用。実画面での扱いを確認。
