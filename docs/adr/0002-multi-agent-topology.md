# ADR-0002: マルチエージェント・トポロジと音声/推論の二層分離

- ステータス: Accepted
- 日付: 2026-06-14

## コンテキスト
要件インタビューには「自然で低レイテンシな会話」と「抜け漏れ・矛盾を検知する深い推論」の双方が必要。
両者を 1 つの LLM 呼び出しに詰め込むと、レイテンシか品質のどちらかが犠牲になる。

## 決定
2 層に分離する。

1. **Voice Agent（Gemini Live, speech-to-speech）**: 会話の主。即応・barge-in・相槌を担う。
2. **ADK Agent Team**: 分析の頭脳。会話の区切りで `analyze_requirements` ツールとして呼ばれる。

ADK チーム内部のトポロジ:
- `Interview Lead Agent`（root）が会話履歴・確定要件から「次の1問」を計画。
- 専門 **sub-agent**（非機能要件 / スコープ&優先度 / 矛盾&抜け検知）に協調的に委譲（中井悦司氏の subagent 思想）。
- Voice Agent から ADK を呼ぶ境界は **agent-as-a-tool**（佐藤一憲氏の思想）。

→ 「subagent 協調」と「agent-as-a-tool」を**意図的に使い分ける**ことで、両審査員の評価軸に整合させる。

## 根拠
- speech-to-speech は STT/TTS 往復より遅延が小さく、自然な相槌・割り込みが可能。
- 分析を別層にすることで、推論モデル（より大きい Gemini）と会話モデル（Live）を独立に最適化できる。
- ステートレスな Voice Agent ＋ 外部永続化により Cloud Run でスケールできる。

## 影響
- 会話の区切り（ターン終了・一定トークン）で ADK を非同期に呼ぶ設計が必要。
- ADK の出力（次の問い・未解決論点）を Voice Agent のコンテキストに戻す経路が必要。
- レイテンシ予算: 会話往復 < 1.5s、ADK 分析は会話を止めない非同期実行。
