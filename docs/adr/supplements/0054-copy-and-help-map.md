## 総括メモ（横断調整・実装時の注意）

- **単一文言**: UI 文言はモードで分岐しない。ADR-0032 の end_user モードはエージェント実行時挙動
  （プロンプト語彙・grounding 出力制御）としてのみ残り、静的 UI 文言は分岐しない。
- **ヘルプ本文の集約**: 同一用語（前提リポジトリ・確信度・発言者・索引・出力フォーマット・メンバー招待/会話リンク）は
  複数画面で再掲される。`lib/help/` の**単一辞書**に一度だけ定義し、各画面はキー参照する。
- **共有ラベル定数（S0）**: 検知（食い違い/確認したい点/あいまい）・分類（機能/使い心地/前提/範囲/確認中）・
  優先度（ぜひ必要/あると助かる/できれば/今回は見送り）・役割 は `lib/realtime/mapping.ts`・`lib/audience.ts` に集約。
  S0 でここを単一文言化し各画面へ波及。
- **backend 由来（S6）**: 出力フォーマット既定テンプレの `絵巻`/`深掘り`/`セッション` は
  `packages/sanba_shared/.../output_formats.py`。web と別 PR（共有パッケージ）で対応。
- **実装時に確認が残る項目**: 選択肢観点 `どう並ぶ？(how)` の意味（暫定「位置づけ」）、`ChatBubble` グリフ `産`/`企` の中立化（P2）。
