# ADR-0050: ドキュメントの配置方針とライフサイクル（AI 駆動開発前提）

- ステータス: Accepted
- 日付: 2026-07-07
- 関連: [ADR-0009](0009-local-compose-split-and-cost.md)（環境変数の用途別再編＝単一の正の思想）/
  [CLAUDE.md](../../CLAUDE.md)（AI コーディング規約）/ [docs/README.md](../README.md)（ドキュメント入口）

## コンテキスト

ドキュメントが ~80 本・5,000 行超に育ち、整備の意図（Diátaxis・ADR 規律・AS-IS↔TO-BE 分離・
ユビキタス言語）は高いが、量産に索引・ライフサイクル管理・単一の正の徹底が追いつかず負債が出ていた。
本リポジトリは AI コーディングエージェント（Claude Code / Codex 等）が**毎セッション規約を読み、
grep でドキュメントを辿り、古い記述を現行仕様と誤読しうる**前提で運用する。したがって配置方針は
「人間だけが読む前提」ではなく「エージェントも一次利用者」という前提で定義し直す必要がある。

主な症状（是正前）:

- **索引の陳腐化**: `docs/README.md` の ADR 一覧が 15/48 本しか無く、`docs/design/`（18 本）や
  `runbooks/`・`automation/` が入口から辿れない。
- **識別子の破綻**: 並行ブランチのマージで ADR 番号が衝突（0042×3・0044×2）。
- **一時文書と恒久文書の混在**: 実装計画・監査・戦略メモが恒久リファレンスに同居し、
  完了済みの「予定」がエージェントに現行仕様として読まれる。
- **単一の正の破れ**: `CLAUDE.md` が `docs/` の内容を写経、`AGENTS.md` が「同内容」を自称しつつ別物、
  セキュリティ設定・検証コマンドが複数箇所に重複。
- **公開境界の逸脱**: 個人情報を含む戦略メモを PUBLIC リポジトリに配置。

## 決定

ドキュメント配置を次の 9 原則で運用する。

1. **常駐層と参照層を分け、常駐層はルータに徹する。** `CLAUDE.md`/`AGENTS.md` は毎セッションで
   コンテキストに載る「規約（憲法）」＝短く・命令的・安定的に保ち、詳細は持たず `docs/` を指す。
   詳細は `docs/` が唯一の正。**「CLAUDE.md は指す、docs が持つ」**（docs 内容を CLAUDE.md に複製しない）。
2. **エージェント規約は単一の正を 1 つ持ち、他は symlink。** `AGENTS.md` はクロスツール標準、
   `CLAUDE.md` は Claude 固有。二重管理を断つため `CLAUDE.md` を実体、`AGENTS.md` を symlink とする。
3. **モノレポは「最寄りの規約」を活かす（ネスト型）。** app 固有規約が分岐したら各 `apps/*/` に
   短い `CLAUDE.md`/`AGENTS.md` を置ける設計とする。規約が均一な現状は root 一本で運用。
4. **恒久リファレンス（SoT）と一時作業文書を物理的に隔離する。** 実装計画・監査・日付付き検証・
   戦略メモは Diátaxis のリファレンスではなく一時文書。`docs/notes/` に置き、着地したら削除/アーカイブ。
   **「ある doc は恒久 SoT か一時作業ノートのどちらか。混在させず、ステータスで明示する」**。
5. **決定は ADR が唯一の正。機械可読メタ＋索引自動生成。** 制御語彙のステータス
   （Proposed/Accepted/Superseded by ADR-NNNN/Deprecated）を持ち、supersede は新旧両方向を更新。
   索引は `just docs-index` で生成し手動追随の陳腐化を根絶。ADR の付随実測資料は docs 直下でなく
   ADR の傍（`docs/adr/supplements/`）に置く。
6. **AS-IS/TO-BE を分け precedence を明記する。** 設計（TO-BE）と実装解析（AS-IS）は別ファイルとし、
   相互リンクと「ズレたら AS-IS を一次情報とする」規則を各冒頭に置く（`architecture.md`↔`architecture-analysis.md`）。
7. **用語は一箇所で定義し他は参照する。** `docs/reference/ubiquitous-language.md` を用語ハブとし、
   各語は出典（`models.py`・ADR・Figma）へリンクする。
8. **公開境界を配置に組み込む。** PUBLIC リポジトリに個人情報・内部戦略・秘匿情報を置かない。
9. **機械可読な導線を 1 つに保つ。** すべての doc が `docs/README.md` から 2 ホップ以内で辿れること。
   `CLAUDE.md` は「docs の入口は `docs/README.md`」と 1 行で指す。

参照層のディレクトリは **Diátaxis を明示ディレクトリ**にする（索引ラベルではなく物理配置で分ける）:

```
docs/
  README.md          入口（4 区分の索引 + 全 ADR 索引=生成）
  tutorials/         学ぶ（手を動かして動く状態まで）
  how-to/            課題を解く（デプロイ・ローカル起動・MCP 等の手順）
  reference/         確認する（アーキ・用語・契約・画面票・セキュリティ方針）
  explanation/       理解する（ロードマップ・設計背景・ペルソナ）
  adr/               設計判断の唯一の正（+ README=索引・supplements/）
  notes/             一時作業文書（非 SoT・完了後は削除/アーカイブ）
```

## 検討したが採用しなかった選択肢

- **Diátaxis を索引ラベルだけで表現し、ディレクトリは現状維持**: 却下。トポロジ的クラスタ（design/ 等）に
  恒久 reference と一時計画が混在し続け、原則 4 が担保できない。物理配置で「区分」と「ライフサイクル」を
  分離する方が、人間にもエージェントにも探索コストが低い。
- **`AGENTS.md` を実体・`CLAUDE.md` を symlink**: どちらでも二重管理は解消するが、本リポジトリの主エージェントが
  Claude Code であり、規約本文の履歴も `CLAUDE.md` にあるため実体を `CLAUDE.md` に置く。
- **一時文書もすべて ADR に統合**: 却下。実装計画・監査は決定ではなく作業の記録で、ADR（不変の決定記録）とは
  性質が違う。ADR に混ぜると決定の一覧性が落ちる。

## 影響

- `docs/` を上記 Diátaxis 構成へ移行し、全内部リンクを追随させる。移行に伴うリンク切れは
  リンク検証（`just docs-index` 併設のチェック）で担保する。
- 索引 `docs/README.md` と `docs/adr/README.md` を生成対象とし、`just docs-index` を追加する。
  以後、ADR を追加・supersede しても索引が自動追随する。
- `notes/` の文書は「着地したら消す」運用とし、恒久リファレンスへ昇格すべき内容が生じたら
  `reference/`・`explanation/` へ移してから notes 側を削除する。
- 本方針は今後のすべてのドキュメント追加に適用する（人間・AI エージェント共通の運用規約）。
