# ADR-0011: Figma MCP による UI/UX デザインループ

- ステータス: Accepted
- 日付: 2026-06-16
- 改訂: デザインループの方向（当初 design→code 主体）は [ADR-0047](0047-app-is-canonical-figma-follows.md)
  で反転（`apps/web` 実装を正本とし Figma は追従）。MCP による Figma 連携ツール自体は継続。

## コンテキスト

機能実装時に、UI/UX を Figma 上で検討・実装・修正まで一気通貫で回したい。
SANBA は web フロント（`apps/web`, Next.js）を持ち、要件定義の解像度を上げるプロダクトである以上、
「要件 → 画面イメージ → コード」を地続きにできると価値が高い。

AI コーディングエージェント（Claude Code 等）から Figma を操作する標準的な手段は MCP である。
Figma の MCP には 2 系統あり、どちらを採用するかが論点になる。

1. **ローカル（Dev Mode）MCP サーバ**: Figma デスクトップアプリが `http://127.0.0.1:3845` で待ち受ける。
2. **リモート（ホスト型）MCP サーバ**: `https://mcp.figma.com/mcp`（Streamable HTTP + OAuth）。

SANBA の開発は Claude Code on the web / クラウド実行環境でも回す。この環境にはデスクトップアプリが
存在せず、ローカル版は `127.0.0.1` を private/reserved IP として解決できず接続失敗する（実測で確認済み）。
したがって「インターネット越しに MCP で扱う」という要件にはリモート版が必須。

## 決定

- **Figma リモート MCP サーバ `https://mcp.figma.com/mcp` を採用**する。
- チーム共有設定としてリポジトリ直下に `.mcp.json` を置く（`type: http`, OAuth はクライアント側フロー）。
  - 認証は OAuth のためトークンをコミットしない。各自が初回利用時に `/mcp` から認可する。
- 用途は双方向:
  - **デザイン → コード**: `get_design_context` / `get_screenshot` / `get_metadata` で Figma の意匠を
    `apps/web` の実装に落とす。
  - **コード → デザイン**: `use_figma` 系で実装意図やコンポーネントを Figma に書き戻し／同期する。
- ローカル版は禁止しない。デスクトップ環境で速度・オフラインを重視する開発者は各自の
  ユーザ設定（`~/.claude.json` 等）でローカル版を上書きしてよい。**リポジトリ既定はリモート版**とする。

## 理由 / 検討した代替案

- **リモート版を既定**: クラウド/web セッションでも動く唯一の選択肢。OAuth で個人トークンの平文管理を避けられる。
  CLAUDE.md「本番志向（どの環境でも動く）」「シークレットをコミットしない」に整合。
- **ローカル版を既定にする案**: 却下。クラウド実行環境で動かず、SANBA の開発フロー（web セッション）と矛盾する。
- **PAT（個人アクセストークン）でのカスタム連携**: 却下。トークン管理コストとリーク面が増える割に、
  公式リモート MCP の OAuth で代替できる。

## 影響 / フォローアップ

- 観測性: MCP 経由のデザイン取得・生成は LLM/ツール呼び出しトレースに乗る。大きなアセット取得は
  レイテンシ要因になり得るため、頻用するなら Langfuse 上で所要時間を観察する。
- セキュリティ: `.mcp.json` の MCP サーバは利用者が初回承認する前提。`/security-review` の対象に MCP 追加を含める。
  社内/機密の Figma ファイルを AI に渡す範囲は人間が判断する（成果物責任は人間）。
- テスト: デザイン取得自体は外部 I/O のため CI 単体テストの対象外。コード化した UI は既存の
  `apps/web` の lint / `tsc --noEmit` / E2E（Playwright）で担保する。
- ドキュメント: 使い方は `docs/how-to/figma-mcp.md` に記載。
- このリポジトリの Claude Code カスタム設定（`.claude/`）とは独立。`.mcp.json` は CLI / web 双方が読む。
