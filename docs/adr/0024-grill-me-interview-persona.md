# ADR-0024: grill-me 流の問い詰めをインタビューペルソナに採用する

- ステータス: Accepted
- 日付: 2026-06-29

## コンテキスト
SANBA(産婆術)の本質は「相手の中にある答えを問いで引き出す」ことにある。初期の音声プロンプトは
「一問一答 + 推奨回答」という grill-me の肝を簡易に取り込んでいたが、本家(mattpocock 版)が持つ
- イエスマンにならず破綻点を率直に指摘する
- ディシジョンツリーを枝ごとに、依存関係の順に一つずつ解消する
- 決定を読み返して確認してから記録する
- 失敗モード・矛盾・曖昧な言葉・トレードオフまで意識的に掘る
といった原則が薄く、表面的な聞き取りに留まりやすかった(issue #267)。`docs/roadmap.md` の Phase 1
「grill-me 流『一問一答 + 推奨回答』を音声で実現」を本格化する必要がある。

なお、開発者がプランを壁打ちするための Claude Code スキル `.claude/skills/grill-me/` は別物であり、
本 ADR が対象とするのは **エンドユーザーと話すプロダクトの音声エージェントのペルソナ**である。両者を
混同しない(前者は開発作業の補助、後者は製品の対話体験)。

## 決定
grill-me 技法を、プロダクトのインタビューエージェントのシステムプロンプトへ本格的に反映する。
既存のマルチエージェント・トポロジ(ADR-0002)は変えず、プロンプト本文の強化で実現する。

適用先(`apps/agent/src/sanba_agent/prompts/interview.py`):

1. `VOICE_AGENT_INSTRUCTIONS`(Gemini Live, 会話の主)— 一問一答・推奨回答に加え、
   枝ごとの解消・率直な指摘・決定の読み返し確認・掘り下げ対象(失敗モード/矛盾/曖昧な語/トレードオフ)・
   「もういい/次へ」はリスクを添えて保留、を明文化。
2. `LEAD_AGENT_INSTRUCTIONS`(次の1問を計画する頭脳)— 出力する1問は「いま開いている枝を1つ解消する」
   問いにし、推奨回答を必須化、表面的なら前提の穴を突く問いを優先。
3. `CONTRADICTION_AGENT_INSTRUCTIONS` — 検出した矛盾はイエスマンにならず直接指摘。

原則の出典は本リポジトリ内 `.claude/skills/grill-me/SKILL.md`(mattpocock 版に忠実な日本語版)。
ただし実運用される正本は `apps/agent/src/sanba_agent/prompts/interview.py` のプロンプト定数であり、
スキルは開発者の壁打ち用、プロンプト定数が製品の対話を駆動するという役割分担を保つ。

## 根拠
- 「答えを引き出す」産婆術の目的に、grill-me の「推奨回答つき一問一答 + 率直な指摘」は直接合致する。
- プロンプト本文への集約は、Langfuse Prompts への同期(コード管理→本番同期)という既存運用に乗る。
- トポロジを変えないため、ADR-0002 のレイテンシ予算・agent-as-a-tool 境界に影響しない。

## 影響
- プロンプト退行を防ぐため、核心原則の存在を assert する軽い回帰テスト
  (`apps/agent/tests/test_prompts.py`)を追加する。LLM 出力の品質評価は引き続き
  `evaluation.py` の LLM-as-a-judge(ADR-0005)が担う。
- 本番では Langfuse Prompts と同期してバージョン管理する(`interview.py` 冒頭の方針どおり)。
