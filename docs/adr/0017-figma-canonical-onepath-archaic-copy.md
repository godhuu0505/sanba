  マクロの一本道・全面古語・a11y 現代語の方針は継続（タブは会話を止めず情報面を横断するサブ切替に限定）。

## コンテキスト

Figma 正本「📱 iPhone 13 Pro（操作フロー・正本）」（fileKey `eI6QvvCEO021zpdMmxr8Iq` / canvas `31:2` / 全12画面 `40:x`）と
`apps/web` 実装の間に、ビジュアル言語と画面構造の乖離があった（epic #138）。

このうち **#139 の金彩 serif テーマ基盤と `components/sanba/*` コンポーネント庫は実装・マージ済み**で、
login/admin も金彩化済み。残るのは実フロー本体（`page.tsx` / `SessionView.tsx` / `DetectionSheet.tsx`）の
結線と新規ビュー（09 アップロード中 / 12 完了）である。

ただし結線に着手する前に、Figma 正本と現行実装で食い違う **2つの方針**を確定する必要があった（#148）。

1. **ナビゲーション**: Figma 正本は戻る `‹` のみの**一本道フロー**（01→…→12）。
   実装（`SessionView.tsx`）は**タブ式**（問答／素材／解析／要件絵巻）で任意に行き来できる。
2. **コピー**: Figma 正本は**古語演出**（「規矩といたしましょう」「相和しませぬ」「奉る」「オーレ！」）。
   実装・screens 票は**現代語・機能ラベル**。

どちらを正とするかが #140/#141/#142/#144 の構造とコピーの前提になる。

## 決定

**Figma 正本を正とする。** 具体的には:

1. **ナビゲーション = 一本道フロー**。`SessionView` のタブ切替を廃止し、会話の自然な進行
   （待機→発話→検知→素材アップロード→解析結果→要件絵巻→完了）で画面が遷移するフロー状態機械にする。
   ヘッダの操作は戻る `‹` のみ。素材／解析／要件絵巻は「タブ」ではなくフロー上のステップとして到達する。
2. **コピー = 全面古語**。見出し・本文・締めの表示テキストを産婆術の世界観に沿う古語で統一する
   （デモ演出としての没入感を最大化する）。
3. **アクセシビリティは現代語で担保（最重要のガードレール）**。表示が古語でも、
   - 操作要素の `aria-label` ／スクリーンリーダー向けテキストは**現代語・機能名**にする
     （例: マイクボタンは見た目が古語でも `aria-label="録音を開始"`）。
   - 状態は色のみに依存させず、**ラベル＋アイコン併記**を維持する（矛盾=緋／抜け=黄土／発話=萌黄／確定=金）。
   - コントラストは WCAG AA を満たす。
   表示の古語化が可読性・操作性・支援技術での理解を損なってはならない。

## 理由 / 検討した代替案

- **一本道を採用**: SANBA の価値は「一問ずつ問答して要件の解像度を上げる」体験そのもので、
  Figma 正本の線形フローはその緩急（§1）を表現する。タブ式は機能実装としては便利だが、
  デモの語り口（産婆術）を弱める。トレードオフとして `SessionView` の状態機械化コストが上がり、
  任意ジャンプができなくなる（戻る `‹` で代替）。#141 の作業範囲は「LiveScreen 結線」から
  「SessionView 全体の一本道フロー化」へ広がる。
- **タブ式維持案**: 却下。実装コストは最小だが Figma 正本と恒久的に乖離し、#147 のドキュメント同期で
  「票を実装に合わせて下げる」ことになり、正本不在の状態が固定化する。
- **全面古語を採用**: プロダクト名の由来（産婆術）と世界観を体験全体で一貫させる。
  ただし**操作の意味は機能から外さない**ため a11y を現代語で別レイヤに分離する（上記ガードレール）。
- **見出し/締めのみ古語・本文現代語案**: 折衷として有力だったが、没入感の一貫性を優先して却下。
  ただし可読性・操作性が実機検証で問題になった場合、本文の一部を現代語へ戻す余地は残す
  （成果物責任は人間。レビューで判断）。
- **全面現代語案**: 却下。最も無難だが Figma 正本の演出価値を捨てることになる。

  に従い、フロー遷移はそのまま realtime イベント（検知／`analysis.progress`／`session.completed`）に駆動される
  ことを確認する（手動でのタブ移動に依存しない）。

- **#148 のクローズ**: 本 ADR をもって #148 の2論点を確定済みとし、issue をクローズする。
