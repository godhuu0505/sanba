## コンテキスト

会話フェーズ（旧 05〜12）の画面を、ステークホルダーとの設計セッションでブラッシュアップした。
要求は「音声で深掘りしつつ 3〜4 択で答える」「いま決まっていること（要件絵巻）・投入済み資料を**会話を止めずに**確認」
「ミュート・会話開始・強制終了」「追加資料の随時アップロード」。

ADR-0017 は navigation を「`SessionView` のタブ切替を**廃止**し一本道に」と決めていた。
しかし上記要求（音声継続のまま情報面を横断）はサブ navigation を必要とし、純粋な一本道では満たせない。

## 決定

会話フェーズに **3 タブのシェル**を導入する。マクロのフローは一本道を維持し、タブはその内側に限定する。

1. **navigation = 一本道（マクロ）＋ 会話フェーズ内 3 タブ（ミクロ）**。
   - マクロ: `03 会話開始 → 04 会話履歴 → 07 判定 → 08 結果`（戻る/終了のみ）。
   - ミクロ: 04 会話履歴 / 05 参考資料 / 06 要件絵巻 を**同一シェルのタブ**にし、
     **音声会話を止めずに**情報面を切り替える。→ **ADR-0017 の「タブ廃止」を会話フェーズについて改訂**。
2. **常時 UI**: ヘッダー（産・問答・REC・経過・終了⏹）／ミニ状況（要件・未確定・資料）／タブ／
   問い＋選択肢ピン／**2 行ボトムバー**を、タブに依らず固定表示。
3. **選択肢の 3 モード**: 最小構成（常時ピン・横スクロール）⇄ 一覧（展開カード）⇄ 詳細（ボトムシート）。
   グラバーで「上=開く/下=閉じる」を一貫。**一覧の行タップ＝即選択**、各行 `詳細 ›` で詳細へ。
   選択肢は動的 3〜4 ＋ 常設 2（「その他＝話す/入力」「わからない・あとで確認＝保留」）。
4. **入力コントロール 2 系統**: 会話（マイク）＝音声入力 ON/OFF、消音＝音声出力 ON/OFF。
   音声 ⇄ テキストは併用（2 行ボトムバーで常時両方）。
5. **素材は 1 パイプライン**: アップロード → **背景解析**（進捗インライン）→ 05 詳細 / 06 要件絵巻へ反映。
   独立の「解析中」画面は持たず 05 に内包。
6. **判定（確定ゲート）**: 回答時・アップロード時に**裏で更新**、**終了押下時**に判定画面で提示。
   未解消 0 件で確定可、1 件でも残れば確定不可。
7. **要件絵巻は閲覧のみ**（編集なし）。
8. **結果出力**: 画面確認＝必須。PDF / Google Drive 連携 / GitHub Issue 作成＝任意。
9. **アートディレクション継続**: 金彩 serif（Noto Serif JP）・全面古語・**a11y は現代語**（ADR-0017 のガードレール維持）。

## 帰結

- **良い**: 音声を止めず「決まっていること」を随時確認でき、抜け・矛盾解消の往復が速い。
  選択肢の確認/即答を両立（行タップ即選択＋詳細）。素材投入で会話が途切れない。
- **トレードオフ**: ADR-0017 の「一本道の純度」を会話フェーズで緩める。タブという復活要素の学習コスト。
  → マクロは一本道のまま・タブは情報面横断に限定し、常時 UI を固定して迷いを抑える。
- **影響先**: `apps/web` の `SessionView` は一本道フロー前提だったため、タブシェルへ再設計が必要。
  `DetectionSheet`/選択肢 UI は 3 モード（最小/一覧/詳細）に拡張。旧 [screens/](../design/screens/) の 05〜10 を 現行設計準拠へ改訂。
- **正本**: Figma フレーム（別行に新規作成、旧 `40:x` は残置）。実装はこの Figma を正とする。

## 代替案

- **純一本道（ADR-0017 のまま）**: 情報確認を「フロー上のステップ」到達に限定。会話を止めずに横断する要求を満たせず却下。
- **選択肢の単一モードのみ**: 確認（詳細）と即答（最小）のどちらかを犠牲にするため却下。3 モードで両立。
- **解析を独立画面**: フローが途切れ「会話を止めない」に反するため、05 内包に変更。
