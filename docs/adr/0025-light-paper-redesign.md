## コンテキスト

これまでの SANBA 体験は「漆黒×金」（`--sanba-bg: #0e0a05` ほか、Figma 正本 31:2）を正とし、
産婆術の世界観を重厚な金彩で表現してきた。一方で次の課題があった。

1. **明るさ**: 黒基調は長時間の壁打ちセッションで重く、日中の利用文脈と乖離する。
2. **テーマ分裂**: admin/login は light（ADR-0014）、SANBA 体験は dark で、二重運用の負債がある。
3. **表情の乏しさ**: 待ち時間・空状態・音声状態の伝達がテキストと色のみで、体温が低い。

ステークホルダーから「サイト全体を白基調へ」「要所に原色に近い色」「棒人間を動かしたい」
（参考: ノーミーツのサイト）という明確な方向の指示があり、デザイン提案 v1（白い紙の上の問答）が承認された。

## 決定

アートディレクションを **「白い紙の上の問答」** に刷新する。情報設計（一本道・3 タブ・ADR-0017/0018）は不変。

1. **色 = 和名の原色パレット**。`--sanba-*` トークンの**名前空間と参照箇所は維持**し、値のみ差し替える。
   - 紙 `#FCFBF7`（下地）／白 `#FFFFFF`（面）／墨 `#221E1A`（線・主要テキスト）
   - 朱 `#C43A20`（主要 CTA・録音・矛盾。白文字で 4.5:1 以上）
   - 瑠璃 `#2A5CDB`（選択・ユーザー発話。新設 `--sanba-select` / `--sanba-select-pale`）
   - 山吹 `#F0A814`（産章・ひらめき・マイク。旧「金」の後継。文字用は `#985C06`）
   - 萌黄 `#5C801F`（音声認識中。旧 `--sanba-speak` を白地向けに暗く調整）
2. **意味論の継承**: 「朱（旧・緋）= 録音・矛盾」「萌黄 = 認識中」「金→山吹 = SANBA の章」を保持する。
   金箔グラデ（`.sanba-gold-gradient`）はベタ塗りの山吹へ、放射グラデ（`.sanba-screen-bg`）は紙色のベタへ。
3. **線 = 墨の手描き線**。カード・ボタン・アイコン・棒人間は墨の線で統一。主要 CTA は
   「墨枠＋ベタ塗りオフセット影」のステッカー様式にし、押せるものを形で示す。
4. **書体 = 太ゴシック中心＋一字だけ明朝**。地の書体を Noto Serif JP（明朝）から
   Noto Sans JP / Zen Kaku Gothic New（見出し）へ。ロゴ円章の「産」一字のみ明朝を残し系譜を継ぐ。
   ADR-0017 の「全面古語・a11y は現代語」の文言ガードレールは**継続**（変えるのは見た目のみ）。
5. **棒人間「サンバさん」= 状態を伝える UI 部品**（`components/sanba/Figure.tsx`）。
   SVG 線画＋CSS キーフレームで `walking / asking / listening / insight / writing` を表現し、
   ホーム・空状態・音声ドックなど**1 画面 1 体まで**に限定。`prefers-reduced-motion` では静止。
6. **テーマ統一**: admin/login の light テーマと地続きになり、`--sanba-*` 名前空間による
   dark/light の分岐運用を解消する方向へ寄せる（旧 dark 値は本 ADR と git 履歴に残す）。

## 影響

- 変更は `apps/web` のみ（トークン値・`components/sanba/*` の意匠・Figure 新設・各画面の微調整）。
  コンポーネントの **props/API は不変**のため、利用側のロジック変更はない。
- コントラストは AA を基準に検証する（墨 16.5:1 / 瑠璃 6.1:1 / 朱 4.6:1 / 山吹文字・鈍色は
  淡色面（`--sanba-surface-strong`）上でも 4.8:1 以上）。
  萌黄・山吹の明るい版はアイコン・大型太字・図形（3:1 基準）に限定する。
- Figma 正本（31:2 系）は漆黒×金のままとなるため、本 ADR 適用後の実装が新たな見た目の正となる。
  Figma 側の更新は別途（ADR-0011 のデザインループで追従）。

## 却下した代替案

- **単純な明度反転（dark → light の機械変換）**: 金は白地で沈み、生成りテキストは読めない。
  色に意味（行動=朱・選択=瑠璃・ひらめき=山吹）を与える再設計を選んだ。
- **トークン名の全面リネーム（`--sanba-gold` → `--sanba-yamabuki` 等）**: 47 ファイル 348 箇所に
  及ぶ機械的変更でレビュー困難になる。名前空間を維持し値を差し替える方が差分が本質に集中する。
- **Lottie 等による棒人間アニメーション**: 依存とバンドルが増える。SVG＋CSS で十分軽量に表現できる。
