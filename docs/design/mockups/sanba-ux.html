● 機能拡張 UI/UX デザイン提案

解像度を、画面で上げていく。

SANBA は音声で対話しながら要件の「抜け漏れ・矛盾」をリアルタイムに検知する壁打ち産婆。 現状の最小UI（参加フォーム＋音声ビジュアライザ）から、準備 → 対話 → 確定 → 振り返りまで 一気通貫で支える体験へ拡張する画面・情報設計をまとめる。

「動くもの」ではなく「届くもの」を。聞く・話す・描く・見るを重ねながら、
人とAIの協働で要件の解像度を上げていく。— 産婆術 (Socratic maieutics)

01 — Design Principles

音声ファースト、認知負荷ミニマム

話すことに集中できるよう「見るべきもの」を絞る。発見（矛盾・抜け）が起きた瞬間だけ画面が静かに前に出る。

P1

声に集中させる

常時は穏やかなビジュアライザと字幕のみ。テキストUIは脳（分析層）の補助に徹し、会話を遮らない。

P2

発見を主役に

矛盾＝赤／抜け＝琥珀。検知した時だけカードがそっと立ち上がり「あ、それ言ってなかった」を演出する。

P3

根拠を必ず添える

確定要件には誰の発話か（出所）・確信度・引用元を常に併記。AIの主張を検証可能にする。

P4

協調を見せる

Lead→専門Agent→再計画の連鎖をトレースで可視化。「ただの薄いボット」ではない必然性を画面で示す。

03 — Information Architecture

情報設計：1本の線で繋ぐ

「会話 → 要件 → 検知 → 確定 → Issue」を分断せず、ユーザーが迷わない直線フローに。準備と振り返りが対話を挟む。

STEP 01ダッシュボード壁打ちを始める／過去セッションと解像度の推移
STEP 02セッション準備ゴール・役割・参考資料(RAG)・同意
STEP 03 ★核ライブ対話音声＋字幕／矛盾・抜けアラート／要件ボード／協調トレース
STEP 04要件サマリMoSCoW構造化／出所・根拠／Issue書き戻し
STEP 05振り返り / 評価検知率 Before/After／エージェント貢献／品質スコア

→ 拡張線：STEP 03 を多人数会議（Phase 2/3）へ自然拡張。話者識別レーンと司会キューを重ねるだけで同じ協調パターンを再利用する。
