# 01 ホーム

- **優先度**: P2（実装済み・差分小）
- **Figma 正本**: node `31:3`（📱 iPhone 13 Pro 操作フロー・正本 / 01 ホーム）

## 目的
入口。「会議の前に5分の問答」という価値を一目で伝え、最短で壁打ち（02 準備）へ送り出す。
実績（今月取り上げた抜け・矛盾）で効果を示し、再訪の動機を作る。

## UI 要素（Figma 由来）
- ステータスバー（9:41 / 5G 100%）— OS 既定、実装対象外。
- ヘッダ: ロゴ「産」+ ワードマーク「SANBA」、下罫線。
- ヒーロー: 見出し「会議の前に、五分の問答を」/ 説明「一問ずつ問いかけ、抜けと矛盾をその場で取り上げます。」
- 主 CTA: **「＋ 壁打ちを始める」** → 02 準備へ遷移。
- 実績カード: ラベル「今月 取り上げた抜け・矛盾」/ 値「38 件 ▲ +41%」（前月比）。
- **過去の要件を見る** 履歴リスト（Figma `99:3`/`99:5`）: ヒーロー下に見出し＋履歴行（標題＋日付＋`›`）。
  各行タップで該当セッション/要件へ遷移（遷移先ルートは別 issue）。空のときは空状態の文言のみ。

## 状態 / 遷移
- 既定: 静的表示。実績値は取得中はスケルトン、失敗時は数値を隠しカード自体を非表示（デモ阻害を避ける）。
- 遷移: CTA タップ → `/`（準備フォーム）。未ログイン時は 02 のログインゲートで受ける。

## 連携
- 実績カード: `GET /api/sessions/stats?period=month`（**新規・P2**）→ `{detections, delta_pct}`。
  未実装の間はカードを非表示にできること（必須要素にしない）。
- 過去の要件を見る 履歴リスト: per-user セッション一覧 API（**新規・別 issue**）→ `{id, title, date}` 配列。
  `SessionHistoryList` は `items` を props で受ける。未接続の間は空状態の文言を出す（#215）。
- データチャネル: なし（ルーム未接続）。

## 受け入れ基準（AC）
- [ ] ヒーローの見出し・説明・CTA が Figma 文言と一致する。
- [ ] CTA タップで 02 準備へ遷移する。
- [ ] 実績 API が無い/失敗でもホームが描画でき、CTA は常に機能する。
- [ ] CTA は thumb-zone（下方）に置けるか、44px 以上のタップ領域を持つ。
- [ ] 実績値は色のみで増減を示さず、`▲`/ラベルを併記する。
- [ ] 過去の要件を見る: ヒーロー下に見出し＋履歴リスト（標題＋日付＋`›`）。各行 44px 以上・`›` で色のみ非依存。空状態の文言を用意。

## 現状差分
- `apps/web/app/page.tsx` L73–124 に相当する導入はあるが、**実績カード**と「壁打ちを始める」の一語 CTA は未整備。
- 現状はログインパネルと準備フォームが同居。01→02 の画面分割は IA 上の整理（必須ではないが正本に合わせる）。

## 根拠
既存実装があり差分が小さいため P2。ただし「会議前5分」の価値訴求と実績の可視化は ADR-0008（コンセプト）の伝達に効く。
