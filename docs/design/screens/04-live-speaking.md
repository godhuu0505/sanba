# 04 会話中・発話中（音声入力）

- **優先度**: P1
- **Figma 正本**: node `31:88`（04 会話中・発話中）

## 目的
ユーザーが話している最中の状態。発話が認識され、裏で複数エージェントが検討していることを**邪魔せず**伝える。

## UI 要素（Figma 由来）
- ヘッダ: 「問答」+「● REC 12:46」（経過時間付き）。
- 中央オーブ + **波形**（音量に反応するバー）。
- 認識状態: 「● 認識中…」。
- 直近発話バブル: 「あなた · 企画」/ 本文（認識テキスト、確定前は淡色）例「関連度順で…いや、さきほど新着順とも申したやも」。
- 検討インジケータ: Figma は「🧠 4体の侍が吟味中」。**実装コピーは機能名で**「🧠 検討中（N体のエージェント）」。

## 状態 / 遷移
- 状態: `phase = listening | recognizing | deliberating`、`agents_active`(検討中の体数)。
- 字幕: `transcript.partial`（確定前・淡色）→ `transcript.final`（確定・通常色）。
- 遷移: 検知が立つと 05 へ（ボトムシートがせり上がる）。検知が無ければ待機 03 に戻る。

## 連携
- データチャネル購読: `status`（agents_active）、`transcript.partial`、`transcript.final`。
- 録音タイマーはクライアント時計でなくセッション開始時刻基準（再接続でズレないよう `session_id` の開始から算出）。
  `POST /api/sessions` レスポンスまたはハイドレーション GET に `started_at`（ISO8601）を含める契約が必要（実装は別PR）。

## 受け入れ基準（AC）
- [ ] 波形が実際の音量入力に反応する（無音時は静止）。
- [ ] `transcript.partial` は淡色、`transcript.final` で確定色に変わり、話者・役割が併記される。
- [ ] 検討インジケータは `agents_active` を反映し、**侍/古語は使わず**機能的な文言で表示する。
- [ ] 「● REC 12:46」の経過時間がセッション開始基準で、リロード後も連続する。

## 現状差分
- 既存は `BarVisualizer` で波形のみ。字幕（partial/final）・話者バブル・検討体数は未実装。
- `transcript.*` / `status` データチャネルの購読を追加（contract 準拠）。

## 根拠
認識と検討の可視化は「薄いボットでない」必然性（ADR-0002/0008）を支える。検知 05 の前段で P1。
