# 05 会話中・検知（音声の結果）

- **優先度**: P0（核となる差別化・未実装）
- **Figma 正本**: node `31:126`（05 会話中・検知）

## 目的
**SANBA の核**。会話中に矛盾／抜けを検知した瞬間、最小割り込みでカード（ボトムシート）をせり上げ、
「あ、それ言ってなかった／さっきと食い違う」を即座に取り上げる。判断は人へ返す（選択肢提示）。

## UI 要素（Figma 由来）
- ヘッダ: 「問答」+「● REC 12:48」。
- オーブ + 状態「◉ 聴いています…」（Figma の古語「産婆、聴いております…」は**現代語に**）。
- 応答バブル「インタビュー統括」（Figma「産婆」）+ 本文（**現代語**。例「その2つは先ほどの発言と食い違います。どちらを優先しますか？」）。
- **検知ボトムシート**（peek→展開）:
  - ラベル「矛盾を検知」（Figma「緋 — 矛盾を検知」/ 色トークン=緋）。抜けの場合「抜けを検知」（色=黄土）。
  - 要約「『関連度順』と『新着順』の両説あり。どちらを基準にしますか？」
  - 選択肢ボタン（任意）: 「関連度順にする」/「新着順にする」。

## 状態 / 遷移
- 入力イベント: `detection.contradiction` / `detection.gap`（[contract](../realtime-contract.md) §3）。
- シート状態: hidden → peek（検知到着）→ expanded（タップ）→ dismissed/resolved。
- 複数検知: スタック（最新を前面、件数バッジ）。`(type,id)` で重複排除、`seq` で順序。
- 解消: 選択肢タップで `user.selection` を送信（[contract §4.5](../realtime-contract.md)）。agent が受信後にツールを呼び出して resolved に遷移（実装は別PR）。

## 連携
- データチャネル購読: `detection.contradiction`, `detection.gap`, `status`。
- 途中参加/リロード時: `GET /api/sessions/{id}/detections?open=1` で未解消検知を復元（P1）。
- 色マッピング: 矛盾=緋（`#D2564B`）/ 抜け=黄土（`#E0A93B`）。**色のみに依存せず**ラベル＋アイコン併記。

## 受け入れ基準（AC）
- [ ] `detection.*` 受信時、ボトムシートが peek 表示され、本文・種別ラベルが正しい色／アイコンで出る。
- [ ] 矛盾・抜けが**ラベルとアイコンでも**判別でき、色覚に依存しない。
- [ ] 選択肢があるときはボタン表示、無いときは要約のみ。
- [ ] 複数検知が時系列（seq）で積まれ、重複が出ない。
- [ ] コピーに古語・侍・産婆を使わず、現代語の機能的表現になっている。
- [ ] `refs`（根拠の発話）をたどれる導線がある（最低でも該当発話を示す）。
