## 目的
素材（モック画像等）から抽出した要件と、**言葉×画の矛盾・抜け**を提示する。マルチモーダル検知（ADR-0004）の成果を
要件へ接続し、09 要件絵巻へ送る。リアルタイム要件リストの中核画面。

## UI 要素（Figma 由来）
- ヘッダ: 戻る `‹` + 「解析結果」。
- 素材プレビュー: 「[ 一覧画面_モック.png ]」。
- **抽出した要件**（チップ）: 「3カラム一覧」/「フィルタUI」/「削除導線」。
- **言葉×画の矛盾**（緋）: 「画面に検索バーが無いが『検索したい』と発言 → 検索バー新設を起票」。
- **抜け**（黄土）: 「並び替えの導線が未配置。空状態の景色も未定義。」
- 主 CTA: **「要件の結果を見る →」** → 09 へ。

## 状態 / 遷移
- 入力イベント: `analysis.visual`(asset_id, extracted, conflicts) + 既存の `requirement.upserted` / `detection.*`。
- リスト構築: 抽出要件チップ・矛盾・抜けを種別ごとに整列。`(type,id)` upsert / `seq` 整列。
- 遷移: CTA → 09 要件絵巻（要件ボード）。

## 連携（ハイドレーション必須）
- **接続/リロード時**: `GET /api/sessions/{id}/requirements`（**P0・新規**）で現在の要件スナップショットを取得 →
  以後 `requirement.upserted` でライブ差分。`GET /api/sessions/{id}/detections?open=1` で未解消検知を補完（P1）。
- データチャネル購読: `analysis.visual`, `requirement.upserted`, `detection.contradiction`, `detection.gap`。
- 出所: 各要件・検知に `source_speaker` / `citations`（どの素材の・どこか）を保持・表示（ADR-0008 #3）。
