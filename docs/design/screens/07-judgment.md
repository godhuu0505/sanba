# 07 判定（確定ゲート）

- **優先度**: P1（確定の見極め）
- **Figma 正本**: `143:2`
- **仕様**: [conversation-experience-v2.md §3,§7](../conversation-experience-v2.md) / [ADR-0018](../adr/0018-conversation-experience-v2.md)

## 目的
要件を「確定してよいか」を**確定ゲート**で見極める。未解消（矛盾/抜け/不明瞭）が残る限り確定させない。
専用レイアウト（タブ共通シェルは使わない）。

## UI 要素（Figma 由来）
- ⚖（金彩）・「見極めの刻にございます」・「確定できるかを検めます」。
- **ゲートカード（緋）**: 「未解消 N 件・確定不可」＋「ひとつでも残れば確定できませぬ」＋未解消項目（矛盾/抜け…）。
- `問答に戻って解く`（主・金）／`未解消のまま終う`（危険・緋枠）。
- 全解消時のバリエーション: 「すべて解けました・確定できます」＋`要件を確定する`（→ 08）。

## 発火タイミング（[§7](../conversation-experience-v2.md)）
- 回答時・ファイルアップロード時に**裏で確定ゲートを更新**（ミニ状況の ⚠未確定 N に反映）。
- **終了押下時**（04 の ⏹ → 終了確認ダイアログ）に本画面で提示。
- ルール: 未解消 **0 件で確定可**、1 件でも残れば確定不可。

## 遷移
`04 ⏹ → 終了確認ダイアログ → 07 判定`。
未解消あり → `問答に戻って解く`（04 へ）／`未解消のまま終う`（08 へ・未確定は確定されない旨を明示）。
全解消 → `要件を確定する`（08 へ）。

## 連携（[realtime-contract](../realtime-contract.md)）
- 未解消の集計は `GET /detections?open=1` と要件確信度から算出。
- 確定で要件をスナップショット（08 の出力対象）。

## 受け入れ基準（AC）
- [ ] 未解消 N 件が**ラベル＋アイコン＋件数**で提示され、内訳（矛盾/抜け…）に飛べる。
- [ ] 未解消 1 件以上では「確定」できず、「戻って解く」または「未解消のまま終う（警告つき）」のみ。
- [ ] 全解消時のみ「要件を確定する」が活性化する。
- [ ] 回答/アップロードでミニ状況の未確定数が裏で更新される。
- [ ] 「未解消のまま終う」は不可逆の影響（確定されない範囲）を明示してから実行する。

## 現状差分
- 確定ゲートは**新規概念**。終了の安全弁として、未解消の可視化と分岐を実装。

## 根拠
ADR-0018（判定=自動更新＋終了時提示・未解消0で確定）。board「判定（確定ゲート）」の画面。
