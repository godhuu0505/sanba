# 03 会話中・待機

- **優先度**: P1
- **Figma 正本**: node `31:64`（03 会話中・待機）

## 目的
ルーム接続直後の静かな待機状態。「話しかけてよい」ことを明確に伝え、音声ファーストの認知負荷ミニマムを体現する。

## UI 要素（Figma 由来）
- ヘッダ: 戻る `‹` + 「問答」+ 録音インジケータ「● REC」。
- 中央オーブ（音声ビジュアライザ、静止/微動）。
- ガイド: 「🎙️ 話しかけてください」/ 補助「マイクを押して発話 →」。
- 下部マイクボタン（thumb-zone）。

## 状態 / 遷移
- 状態: `phase = idle`（[realtime-contract](../realtime-contract.md) の `status` イベント）。
- 遷移: ユーザー発話開始 → 04 発話中（`status.phase = listening`/`recognizing`）。
- 録音インジケータは同意済みのセッションでのみ点灯。

## 連携
- データチャネル購読: `status`（phase=idle/listening）。
- 音声: 既存 LiveKit。マイク権限が未許可ならガイドを権限要求に差し替える。

## 受け入れ基準（AC）
- [ ] 接続完了後、既定で待機表示（「話しかけてください」）になる。
- [ ] `status.phase` の変化でオーブの状態が idle→listening に切り替わる。
- [ ] マイクボタンは 44px 以上で下部に固定。権限未許可時は許可導線を出す。
- [ ] 「● REC」は録音中のみ表示し、同意の事実と矛盾しない。

## 現状差分
- `apps/web/app/page.tsx` の `InterviewView()`（L160 付近）は state 文字列表示のみ。待機の専用レイアウト・ガイド文言は未整備。
- `status` データチャネルの購読が未実装（contract に従い追加）。

## 根拠
核フロー（対話）の入口で、状態遷移の基盤。05/04 と一体で P1。
