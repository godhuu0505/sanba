# 02 準備（テキスト入力フォーム）

- **優先度**: P1
- **Figma 正本**: node `31:23`（02 準備）

## 目的
壁打ちの前提（ゴール・自分の役割・参考資料・同意）を整える。資料を RAG に取り込み「既知の要件」を把握させ、
**重複を避け未確定から問う**ための土台を作る。

## UI 要素（Figma 由来）
- ヘッダ: 戻る `‹` + 「セッション準備」。
- **ゴール（テキスト入力）**: 例「検索機能のリニューアル要件を固めたい」。単一行〜複数行。
- **あなたの役割**: チップ選択 `● 企画(PdM)` / `エンジニア` / `顧客`（単一選択、既定=企画）。
- **参考資料（バイナリ添付）**: 添付済み行「📄 PRD_検索改善.pdf」+ 状態バッジ「✓ 解析済 24chunk」。追加導線は 06 へ。
- 把握サマリ: 「✓ 資料から既知 8 件を把握。重複を避け未確定から問います。」
- **同意ゲート**: 「✓ 録音と AI 処理に同意（30日保持・PIIマスク）」（必須チェック）。
- 主 CTA: **「🎙️ インタビューを始める」**（同意 OFF の間は無効）。

## 状態 / 遷移
- フォーム状態: ゴール文字列 / 役割 / 添付ファイルとその解析状態（未解析→解析中→解析済 Nchunk）/ 同意フラグ。
- CTA 活性条件: 同意=ON（`REQUIRE_CONSENT=true` のとき必須）。ゴール空でも開始は可（任意）。
- 遷移: CTA → セッション作成 → 03 待機（ルーム接続）。

## 連携
- `POST /api/sessions`（既存・認証不要の公開 API）→ `session_id` + 役割別招待トークン。
- `POST /api/sessions/{id}/context`（既存・テキスト）/ `POST /api/sessions/{id}/context/file`（既存・PDF/MD/TXT）。
  添付の「✓ 解析済 Nchunk」はファイル取り込みのレスポンス（chunk 数）を表示する。
- `POST /api/sessions/join`（既存）→ LiveKit 参加トークン → ルーム接続。
- データチャネル: 接続後に購読開始（03 以降）。

## 受け入れ基準（AC）
- [ ] 同意 OFF のとき「インタビューを始める」が無効で、理由（同意が必要）が示される。
- [ ] 役割は単一選択で、既定が「企画(PdM)」。選択値が `POST /api/sessions` の役割に渡る。
- [ ] ファイル添付が解析完了すると「✓ 解析済 Nchunk」が実際の chunk 数で表示される。
- [ ] ゴール未入力でも開始でき、入力時は `POST /api/sessions/{id}/context`（`source_name=goal`）で文脈として取り込まれ、RAG・会話初期文脈に反映される。
- [ ] 保持期間（30日）と PII マスクの文言が表示され、`DATA_RETENTION_DAYS` / `MASK_PII_BEFORE_INDEX` と整合する。
- [ ] CTA タップ後、二重送信できない（処理中は無効化）。

## 現状差分
- `apps/web/app/page.tsx` L77–99 にフォームの原型あり（名前・役割・参考資料テキスト・同意）。
- 差分: ①「ゴール」入力の明確化、② ファイル添付の解析状態バッジ（chunk 数）、③「既知N件を把握」のサマリ表示。
- 「名前」入力は Figma に無い（役割中心）。名前フィールドは省き、役割選択のみとする（Google 認証は別要件として別途検討）。

## 根拠
既存の中核フォーム。準備の質が対話の質を決めるため P1。資料 grounding は ADR-0003、同意は issue #10 と直結。
