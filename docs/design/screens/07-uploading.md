# 07 アップロード中（バイナリ挙動）

- **優先度**: P1
- **Figma 正本**: node `31:205`（07 アップロード中）

## 目的
素材アップロード〜AI 解析の進行を可視化し、会話を止めずに「いま何をしているか」を伝える（二層構造, ADR-0002）。

## UI 要素（Figma 由来）
- ヘッダ: 戻る `‹` + 「画像をアップロード」。
- ファイル行: サムネ +「一覧画面_モック.png」+「2.4 MB · PNG」+ 状態「⏳」。
- 進捗バー +「AI が解析中… 62%（画面要素・画面内テキストを抽出）」。
- ステップチェックリスト: 「✓ 画面領域の検出」/「✓ オンスクリーン文字の OCR」/「● 既存発話との突き合わせ…」。

## 状態 / 遷移
- 状態: アップロード（クライアント→API）→ 解析（API/agent）。`analysis.progress`(pct, stage) を反映。
- 遷移: `pct=100` / 完了イベントで 08 解析結果へ。失敗時はエラー＋再試行。
- 会話継続: 本画面表示中も音声セッションは生きている（非同期・ブロックしない）。

## 連携
- アップロード: `POST /api/sessions/{id}/context/file`（拡張後）。クライアント側の送信進捗はリクエスト進捗で表示可。
- 解析進捗: データチャネル `analysis.progress`（agent/API が publish）。**現状 API はブロッキングで進捗イベント無し**
  → 進捗の段階通知は別PRで追加（要件として明記。無い間は不確定プログレス＝スピナーで代替可）。

## 受け入れ基準（AC）
- [ ] ファイル名・サイズ・形式が実値で表示される。
- [ ] `analysis.progress` を受けて％とステップ（領域検出→OCR→突合）が進む。
- [ ] 進捗イベントが無い環境では不確定インジケータにフォールバックし、固まらない。
- [ ] 解析中も音声会話が継続できる（UI が会話をブロックしない）。
- [ ] 失敗時にエラー理由と再試行導線が出る。

## 現状差分
- Web UI 未実装（送信中は `button.disabled` のみ）。進捗バー・ステップ表示を新規実装。
- 進捗を出すには API/agent 側の段階イベント（`analysis.progress`）追加が前提。

## 根拠
解析の体感を支えるが、進捗イベント基盤の追加を要し核そのものではないため P1。
