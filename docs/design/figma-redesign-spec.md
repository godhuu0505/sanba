## 6. 反映手順（`use_figma`）と検証

1. フォント（Noto Sans JP / Zen Kaku Gothic New / Noto Serif JP）を `loadFontAsync`。未提供時は Noto Sans JP に縮退。
2. フレーム背景 → `#FCFBF7`、旧グラデ塗りを除去。
3. カード/ボタン/チップ/吹き出し/ドックを §3 の様式へ。色は §1 の対応表で機械的に置換。
4. 棒人間 SVG を `Figure.tsx` から移植（`createNodeFromSvg`）し §4 の位置へ。
5. 各フレームを `get_design_context`（`excludeScreenshot: true`）で読み戻し、**背景色・文字色・枠色・書体**が
   新トークンに一致することを構造照合する。最終的なピクセル目視は人間が Figma 上で行う（成果物責任は人間・CLAUDE.md）。

> 本環境の制約: egress ポリシーにより `figma.com` からの画像取得（スクリーンショット PNG）が不可。
> そのため自動検証は「MCP の構造（コード）読み戻し」で行い、見た目の最終確認は Figma 上の人手に委ねる。
