## A. 認証ゲート仕様の検証（明示要件）

> 要件: **「ログインしていれば全画面にアクセスでき、ログインしていなければログイン画面に戻る」**

### 結論

- **「ログイン時は全画面アクセス可」… ほぼ成立。** ログイン済みなら `/`（ホーム→準備→会話）・`/admin` に到達できる。
  ただし `/admin` は **認可（管理者）** を API 側 `ADMIN_EMAILS` で判定し、非管理者は 403 ゲート表示になる（これは認証ではなく認可で正しい）。
- **「未ログイン時はログイン画面に戻る」… 対応済（[#217](https://github.com/godhuu0505/sanba/issues/217) で結線）。**
  `RequireAuth`（`components/RequireAuth.tsx`）の共通判定 `authGate(auth, next)` を保護ルートに集約し、
  未ログイン/解決前は `router.replace('/login?next=...')` で `/login` へ自動リダイレクトする（方針案1=厳密を採用）。
  - 結線済みルート: `/`（`app/page.tsx`）・`/admin`（`app/admin/page.tsx`）・`/settings`（`app/settings/page.tsx`）。
    いずれも `const gate = authGate(auth, "<route>"); if (gate) return gate;` で早期 return する。
  - `app/` 配下に `middleware.ts` は無いが、ルート保護はクライアント側 `authGate` に集約済み。
    旧記述「`RequireAuth` がどのページからも import されていない（＝死蔵）」「ルート保護は実質『無し』」は**現状では誤りで、解消済み**。

### ルート別の実態

| ルート | 未ログイン時の挙動 | 期待（要件） | 判定 |
|---|---|---|---|
| `/login` | ログイン画面（公開） | 公開でよい | OK |
| `/`（01 ホーム / 02 準備 / 03–08 会話フェーズ） | `authGate(auth, "/")` が未ログイン時に `/login?next=/` へ**自動リダイレクト**し、ホームは描画しない（real モード）。dev モードは `AUTH_DEV_BYPASS` に委ね素通し | /login へ戻す | **OK（#217）** |
| `/admin` | `authGate(auth, "/admin")` で未ログインは `/login?next=/admin` へ**自動リダイレクト**。認証後、非管理者は API `ADMIN_EMAILS` 判定で 403 ゲート表示（認可は別レイヤで正しい） | /login へ戻す | **OK（#217）** |
| `/settings` | `authGate(auth, "/settings")` で未ログインは `/login?next=/settings` へ**自動リダイレクト**（認証ゲート配下） | /login へ戻す | **OK（#217/#227）** |
| `/design` | 開発専用（`NODE_ENV==='production'` で `notFound()`） | — | OK |

### 設計意図との衝突（解決済み）

- 初版では `app/page.tsx` のコメントと `app/page.test.tsx` が **ホームを公開ランディングとして意図**しており、
  「**全画面**でリダイレクト」を厳密採用するとこの意図に反する、という判断ポイントがあった。
  - **方針案1（厳密）**: 全保護ルート（`/`, `/admin`）を `RequireAuth` でラップし未ログインは `/login?next=` へ。ホームのランディング性は失う。
  - **方針案2（折衷）**: ホームは公開ランディングのまま、**会話フェーズと `/admin` のみ**保護。
- **採用結果（#217）**: 共通判定 `authGate` を新設し、**方針案1（厳密）**を採用した。`/`・`/admin`・`/settings` を
  `authGate` で保護し、未ログインは `/login?next=` へ自動リダイレクトする。`app/page.test.tsx` も
  「real モードで未ログインなら `/login?next=/` へリダイレクトしホームを描画しない」へ更新済みで、ホームのランディング性は持たせない方針に確定した。
