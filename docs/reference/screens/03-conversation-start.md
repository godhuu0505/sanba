## 目的
準備（02）から会話フェーズへ橋渡しする。**接続とマイク許可を確実に取り**、失敗時は理由提示＋復帰導線を出す。
専用レイアウト（タブ共通シェルは使わない）。

## 状態と UI 要素（Figma 由来）

|---|---|---|
| 03-0 開始前 | 産オーブ（金彩）・「支度、相整いまして…」・準備サマリ（ゴール/役割/参考資料）・`🎙 問答を始める`・「マイクを使用します」注記 | 開始 |

## 遷移
`02 準備 →[会話を始める]→ 03-0 → 03-2 許可 →（許可）03-1 接続 →（成功）04 会話履歴`。
許可拒否 / 接続失敗 → 03-3（設定導線・再試行で復帰）。開始時のテキスト専用モードは無い
（セッション中のテキスト入力は 04 会話画面で提供する）。
タイムアウト・再接続中は 03-1 のバリエーション。

## 連携
- 接続: LiveKit ルーム参加（[ADR-0006](../../adr/0006-cloudrun-and-livekit.md)）。トークン発行は API（[realtime-contract](../realtime-contract.md)）。
- 権限: 端末マイク権限。拒否時は OS 設定への誘導文言（再要求は OS 制約に従う）。
- 認証: 本人確認は Google ログイン（[ADR-0012](../../adr/0012-google-login.md)）。**未ログインで保護ルート（`/` 準備・会話フェーズ）へアクセスした場合は `/login` へリダイレクト**（401＝未ログイン）。ログイン成功後は**元の遷移先（準備/会話）へ復帰**。管理者でない場合は 403（94 アクセスゲート）。
- セッション中のテキスト発話送信は `user.text`（[realtime-contract §4.5](../realtime-contract.md) / [#185](https://github.com/godhuu0505/sanba/issues/185)）。開始時のテキスト専用モードは提供しない。

## 受け入れ基準（AC）
- [x] 03-0 にゴール/役割/参考資料のサマリが出る（02 の入力を引き継ぐ）。
- [x] 許可前に必ず 03-2 モーダルで**理由提示**してから OS プロンプトを呼ぶ。
- [x] 拒否/失敗で 03-3 に遷移し、**設定導線・再試行**の復帰導線が出る。
- [x] 接続中はキャンセル可能。タイムアウト時に再試行導線が出る。
- [x] **未ログインで保護ルートに来ると `/login` へリダイレクト**され、ログイン成功後に元の画面へ復帰する。
- [x] 表示は古語でも `aria-label`・状態はラベル＋アイコン併記（色のみ依存しない）。

## 現状差分
- 03-0 開始前サマリ / 03-1 接続中 / 03-3 失敗系は実装済み（`components/ConversationStart.tsx`）。
- **03-2 録音許可モーダル**（`MicPermissionModal`）を実装し、03-0 の「🎙 問答を始める」→ OS プロンプト前にアプリ内モーダルで理由提示する経路を追加（#216）。
- 03-3 の「設定を開いて許可する」を**第一 CTA のボタン**に格上げ（マイク失敗時のみ）。押下で手順ガイドを同画面に展開する（Web から設定を直接開けないため文言フォールバック）。接続失敗時は設定導線を出さない。
