# 08 結果（要件産婆結果）

- **優先度**: P1（締め・成果物の受け渡し）
- **Figma 正本**: `144:2`
- **仕様**: [conversation-experience.md §3,§7](../conversation-experience.md) / [ADR-0018](../../adr/0018-conversation-experience.md)

## 目的
確定した要件を「産まれた成果物」として受け取る。**画面で確認＝必須**、外部出力は任意で提供。
専用レイアウト（締め演出）。

## UI 要素（Figma 由来）
- 産オーブ（金彩・発光）・「オーレ！ 要件、産まれました」・「確定要件 N 件（Must / Should / Could）」。
- **確定要件カード**: 区分バッジ＋本文＋`✓` ・「ほか N 件 ・ タップで全文 ›」。
- 主操作 `この絵巻を画面で確認する`（既定・必須）。
- **書き出す（任意）**: `📄 PDF` / `☁ Drive` / `🐙 Issue`。
- `新しい問答を始める`。

## 出力（[§7](../conversation-experience.md)）
| 出力 | 位置づけ |
|---|---|
| 画面で確認 | **必須**（既定動線） |
| PDF 出力 / Google Drive 連携 / GitHub Issue 作成 | **任意**（あると良い・段階導入） |

## 遷移
08 は2パターンで到達する:
- **確定終了**（全解消）: `07 → 要件を確定する → 08`。確定要件を成果物として提示。
- **未確定終了**（未解消あり）: `07 → 未解消のまま終う → 08`。「未解消 N 件が残った暫定出力」として別コピー・別スタイルで提示し、確定と区別する。
`新しい問答を始める`→ 01/02 へ。出力は各コネクタへ。

## 連携
- 確定要件のスナップショット（07 で確定）を表示・書き出し。スナップショット書き込み契約は未定義（[#186](https://github.com/godhuu0505/sanba/issues/186)）。
- 外部出力は外部コネクタ（[ADR-0007](../../adr/0007-external-connectors.md)）: Drive / GitHub Issue。PDF はレンダリング。

## 受け入れ基準（AC）
- [ ] 確定要件が MoSCoW で**画面上で全件確認**できる（必須動線）。
- [ ] **未確定終了**の場合は「暫定出力」である旨を明示し、確定終了のコピー（「要件、産まれました」等）と区別する。
- [ ] PDF / Drive / Issue は**任意**として提示され、未実装でも本線（画面確認）が成立する。
- [ ] 「新しい問答を始める」で次セッションへ導線がある。
- [ ] 締めコピーは古語演出でも、操作の a11y ラベルは現代語・機能名。
